# 产品规格

## 功能特性

1. **用户输入**：用户可输入当前提问的时间、地点、天气、心情、身边的人等信息。

2. 模式选择：提供惯性模式和随机模式供用户选择。

   - **惯性模式**：AI 判定新提问在 ragflow 中的最接近值并返回。
   - **随机模式**：先从 ragflow 中随机一个值，AI 自定义一个吃的，判断与随机值是否接近，相近选新值，否则选旧值。

   

3. **数据记录**：每次用户提问记录到 ragflow 中。

4. **后端服务**：基于 Java、springboot、springai、ragflow、MySQL 和 Redis 搭建，处理业务逻辑、数据存储与检索。

5. **前端展示**：采用 vue 构建交互界面，方便用户操作。

## 技术细节

1. 后端：

   - Java 作为开发语言，springboot 构建基础框架。
   - springai 实现 AI 相关逻辑。
   - ragflow 用于数据记录与匹配。
   - MySQL 存储用户提问、匹配结果等数据。
   - Redis 缓存常用数据提高响应速度。

   

2. **前端**：vue 组件化开发，与后端 API 交互获取和展示数据。

# 项目计划

## 第一阶段（1 - 2 周）：需求细化与设计

1. 完成详细的功能设计文档。
2. 设计数据库表结构。
3. 规划前后端接口。

## 第二阶段（3 - 6 周）：后端开发

1. 搭建 springboot 基础项目。
2. 实现基于 springai 和 ragflow 的 AI 判断逻辑。
3. 完成 MySQL 和 Redis 的数据交互。

## 第三阶段（7 - 9 周）：前端开发

1. 搭建 vue 项目框架。
2. 实现用户输入界面、模式选择及结果展示。
3. 完成与后端接口的联调。

## 第四阶段（10 - 11 周）：测试与优化

1. 进行功能、性能测试。
2. 修复测试中发现的问题。

## 第五阶段（12 周）：上线部署

1. 将应用部署到生产环境。
2. 进行上线后的验证。
