## 三、核心功能流程

### 3.1 交互式对话流程

```
用户发起推荐请求
      │
      ▼
后端初始化会话
├─ 随机总提问次数 N（5-10次）
├─ 初始化7个参数状态
├─ 创建会话存储到Redis
└─ 调用AI开始对话
      │
      ▼
AI返回第一次提问
{"type":"question", "param":"time", "content":"请问您现在是什么时间想吃东西呢？"}
      │
      ▼
后端校验 → 关键词匹配 → 返回前端
      │
      ▼
用户回答
      │
      ▼
AI校验回答有效性
├─ 有效 → {"type":"chat", "content":"好的，收到啦！"} → 记录数据库 → 继续提问
└─ 无效 → {"type":"2question", "param":"time", "content":"请再说清楚一点"} → 追问
      │
      ▼
循环直到收集完成或达到提问次数
      │
      ▼
进入推荐阶段
```

### 3.2 标签类型定义

| type值 | 含义 | 计入总次数 | 使用场景 |
|--------|------|-----------|----------|
| `question` | 正常提问 | ✅ +1 | 首次询问某参数 |
| `2question` | 追问 | ❌ | 用户回答不合格时追问 |
| `chat` | 确认/闲聊 | ❌ | 确认用户回答、自由发挥 |
| `interrupt` | 打断回复 | ❌ | 用户抢话时回复 |
| `recommend` | 推荐结果 | ❌ | 最终推荐 |
| `system` | 系统消息 | ❌ | 系统通知 |

### 3.3 参数收集规则

**必选参数（7个）：**
- 时间、地点、天气、心情、同行人、预算、口味偏好

**可选参数（自由发挥阶段）：**
- 饮食禁忌/过敏
- 最近偏好
- 健康需求

**提问规则：**
- 总提问次数：随机5-10次
- 单个参数追问上限：2次
- 第5次提问后进入自由发挥阶段

## 四、WebSocket通信设计

### 4.1 消息格式

**客户端发送：**

```json
{
  "action": "start" | "answer" | "complete",
  "sessionId": "xxx",
  "content": "用户回答内容"
}
```

**服务端推送：**

```json
{
  "type": "question" | "2question" | "chat" | "interrupt" | "recommend" | "system",
  "param": "time",
  "content": "消息内容",
  "progress": {
    "current": 3,
    "total": 7,
    "collected": ["time", "weather"]
  }
}
```

### 4.2 抢话处理机制

```
时间线示例：
T1: 用户发送 "我想吃东西"
T2: AI正在处理...
T3: 用户发送 "我在公司"    → 存入Redis pending队列
T4: 用户发送 "现在是晚上"  → 存入Redis pending队列
T5: AI处理完成，推送结果
T6: 检测到pending队列有消息
T7: 汇总发送给AI，type=interrupt
T8: AI回复 "你说话太快啦，让我先想想~"
T9: 继续之前未完成的流程
```

**状态管理：**

```java
ConversationState {
  aiProcessing: boolean          // AI是否正在处理
  sendingToFrontend: boolean     // 是否正在推送到前端
  pendingMessages: List<String>  // 用户抢话消息队列
  interruptCount: int            // 打断次数（上限10）
  pendingResponse: Message       // 待推送的消息（被打断时保存）
}
```

### 4.3 消息队列处理

- AI异步生成消息，即时推送
- 使用消息队列保证消息顺序
- 无人为延迟，保证流畅体验

## 五、随机模式相似度判断

### 5.1 判断流程

```
用户输入 → 从RagFlow随机获取旧值 → AI生成新值 → AI判断相似度
                                              │
                                              ▼
                              ┌─────────────────────────────┐
                              │ 相似度 < 0.7 → 选旧值       │
                              │ 相似度 >= 0.7 → 选新值      │
                              └─────────────────────────────┘
```

### 5.2 相似度判断提示词

```
请判断以下两种食物的相似程度：
食物A：{food1}
食物B：{food2}

评估维度：
- 主要食材是否相似
- 口味类型是否相近
- 烹饪方式是否类似
- 用餐场景是否相同

返回0-1之间的数字：
- 0.0-0.3：非常不相似
- 0.3-0.5：不太相似
- 0.5-0.7：有一定相似度
- 0.7-1.0：非常相似

只返回数字，不要任何解释。
```

## 六、数据库设计

### 6.1 表结构

```sql
-- 用户会话表
CREATE TABLE conversation_session (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) UNIQUE NOT NULL,
    user_id BIGINT,
    total_questions INT,
    current_question_count INT,
    interrupt_count INT DEFAULT 0,
    mode VARCHAR(20),  -- inertia/random
    status VARCHAR(20),
    created_at DATETIME,
    completed_at DATETIME
);

-- 问答记录表
CREATE TABLE qa_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) NOT NULL,
    question_type VARCHAR(20),  -- question/chat/2question
    param_name VARCHAR(50),
    ai_question TEXT,
    user_answer TEXT,
    is_valid BOOLEAN,
    question_order INT,
    created_at DATETIME,
    INDEX idx_session (session_id)
);

-- 参数收集结果表
CREATE TABLE collected_params (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) NOT NULL,
    param_name VARCHAR(50),
    param_value TEXT,
    param_type VARCHAR(20),  -- required/optional
    collected_at DATETIME,
    UNIQUE KEY uk_session_param (session_id, param_name)
);

-- 推荐结果表
CREATE TABLE recommendation_result (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    session_id VARCHAR(64) NOT NULL,
    mode VARCHAR(20),
    food_name VARCHAR(100),
    old_food VARCHAR(100),  -- 随机模式的旧值
    similarity_score DECIMAL(3,2),
    reason TEXT,
    created_at DATETIME
);
```

## 七、API接口设计

### 7.1 REST API

```
POST /api/conversation/start
- 启动新的对话会话
- 返回：sessionId, WebSocket连接地址

GET /api/conversation/status/{sessionId}
- 获取会话状态
- 返回：已收集参数、当前阶段、进度

POST /api/conversation/complete/{sessionId}
- 手动结束收集，直接进入推荐
- 返回：推荐结果

GET /api/conversation/history/{sessionId}
- 获取对话历史
- 返回：问答记录列表
```

### 7.2 WebSocket端点

```
ws://host/ws/conversation/{sessionId}

消息类型：
- start: 开始对话
- answer: 用户回答
- complete: 手动完成
```

## 十一、关键决策摘要

| 项目 | 决策 |
|------|------|
| 后端框架 | Spring Boot 3.4.x + Java 21 |
| 前端框架 | Vue 3 + Vite + Vant 4（移动端优先） |
| 通信方式 | WebSocket |
| 标签类型 | question / 2question / chat / interrupt |
| 计数规则 | question+1，其他不计；interrupt单独计（上限10） |
| 相似度计算 | AI语义判断，阈值0.7 |
| 记忆功能 | 单次对话记忆 |
| 抢话处理 | pending队列 → 汇总 → interrupt回复 → 继续原流程 |
